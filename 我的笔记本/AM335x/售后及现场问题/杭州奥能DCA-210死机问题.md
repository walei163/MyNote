# 杭州奥能DCA-210 V2.6版本装置现场运行死机的问题

> [!caution]
>
> <font face="微软雅黑" color=yellow>**问题1：**</font><font face="微软雅黑" color=yellow>**奥能在现场，发现DCA-210 V2.6版本装置运行出现死机，系统打印的日志，总是提示出现`I2C0`操作RTC时出现ack丢失的现象。**</font>

因为目前暂时无法确认问题的原因，先做以下几个方面的措施：

1、更换外接12V电源。

2、外部添加截取日志的工具。

3、去掉`crond`服务，不再定时写RTC。

4、修改4G自动拨号程序，不再周期性写RTC。

5、担心`I2C0`操作问题会影响到电源管理芯片：`TPS65910`，修改脚本：`rc.user`，添加以下代码：

```sh
echo Setting CPU frequncy to 800MHz...
echo userspace > /sys/devices/system/cpu/cpu0/cpufreq/scaling_governor
echo 800000 > /sys/bus/cpu/devices/cpu0/cpufreq/scaling_setspeed
cat /sys/devices/system/cpu/cpu0/cpufreq/cpuinfo_cur_freq
```

这样使得系统一直以固定频率运行，内核不再动态调整频率和电压。
